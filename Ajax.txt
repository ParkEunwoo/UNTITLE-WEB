 
- 에이잭스 _ AJAX (Asynchronous JavaScript and XML) -

 1. 정의
  - 비동기식 자바스크립트  XML ( JavaScript 와 XML을 이용한 비동기적 정보 교환 기법 )
   + XML이라지만 거의 JSON 데이터를 다룬다 함
  - WEB2.0 기반의 기술
  - 특정한 기술이 아니라 대화식 웹 애플리케이션의 제작을 위해 사용하는 기술의 묶음
   + 표현 정보를 위한 HTML 과 CSS
   + 동적인 화면 출력 및 표시 정보와 상호작용을 위한 DOM, 자바스크립트
   + 웹 서버와 비동기적으로 데이터를 교환하고 조작하기 위한 XML, XSLT, XMLHttpRequest
  - HTML 만으로 어려운 다양한 작업을 웹페이지에서 구현하여 유저가 웹페이지와 자유롭게 상호작용할 수 있도록 하는 기술
  - 별도 프로그램 없이 웹페이지를 다시 로딩하지 않고 메뉴 등 화면상의 객체를 자유롭게 움직이고 다룰 수 있음
  - 비슷한 기능으로 액티브 X나 플래시에 비해 가볍고 속도가 빨라 차세대 웹 기술로 각광
  

 2. 기존 기술과 차이점
  - 기존의 웹 애플리케이션
   + 브라우저에서 폼을 채우고서버에 제출하면 서버는 요청된 내용에 따라서 데이터를 가공해 새로운 웹페이지를 작성 및 응답
   + 이는 결과적으로 중복되는 HTML 코드를 다시 전송 받으면서 많은 대역폭을 낭비함

  - AJAX
   + 필요한 데이터만을 웹서버에 요청해서 받은 후 클라이언트에서 데이터에 대한 처리를 할 수 있음
   + 브라우저와 서버간 교환되는 데이터량과 서버의 데이터 처리량도 줄어들어 응답성이 좋아짐

 3. 원리 / 과정
  - 브라우저가 사이트에 접속하면 서버는 사이트의 기본 구조를 담은 ‘템플릿’을 전달
  - 브라우저는 수신받은 템플릿 HTML과 CSS를 해석해 화면의 기본 모양을 그림
  - 계속해서 서버는 데이터의 요청 방식과 수신받은 데이터를 어떻게 가공해야 하는지를 기술한
    자바스크립트 파일을 전달
  - 브라우저는 자바스크립트 파일을 해석해서 파일에 기술된 방식대로 서버에 추가 데이터를 요청
  - 서버는 순수 데이터를 응답으로 되돌려줌
  - 브라우저는 수신한 데이터를 해석하여 템플릿의 적절한 위치에 삽입 (삽입 외에 변경, 삭제 가능)  

 4. 장점
  - 페이지 이동없이 고속으로 화면을 전환 가능
  - 서버 처리를 기다리지 않고, 비동기 요청이 가능
  - 수신하는 데이터 양을 줄일 수 있고, 클라이언트에게 처리를 위임

 5. 단점
  -  Ajax를 사용하지 않는 브라우저에 대해 문제가 있음
   + 주로 구버전 : 오페라 7 이하, 익스프로러 4.0 이하, 1997년 이전 브라우저
   + 텍스트 기반의 브라우저 링크스, w3m, 시각장애인을 위한 브라우저
  - HTTP 클라이언트의 기능이 한정
  - 페이지 이동이 없는 통신으로 보안상의 문제
  - 스크립트로 작성되어 디버깅이 용이하지 않음
  - 요청을 남발하면 역으로 서버 부하가 늘 수 있음
  - 동일-출처 정책으로 인해 다른 도메인과는 통신이 불가능
  - 자바스크립트를 해석하지 못하는 검색 엔진에서는 내용이 검색되지 않음
  - JavaScript를 사용해야만 함
  - 이 방식은 비동기 데이터 전송 기술이지만 양방향이 아니란 점에서 문제점이 있기에 이 기술을 토대로
  HTTP 프로토콜에 약간의 변형을 가한 웹 소켓이란 새로운 기술과 실시간 통신 기술인 Long pooling 기술이 생겨났음
