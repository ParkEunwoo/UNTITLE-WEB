http
hyper text transfer protocol

웹 클라이언트와 웹 서버 간의 요청과 응답이 http 프로토콜로 이루어진다

이때 웹 서버가 제공하는 것을 리소스라고 하고 리소스의 위치를 URL을 이용해서 표현한다

리소스들에는 타입이 주어지는데 그 타입에 MIME이라는 데이터 포맷 라벨을 붙인다
주타입/부타입 으로 이루어져 있다

html 로 작성된 텍스트 문서 -> text/html
jpeg 이미지                     -> image/jpeg
png 이미지                     -> image/png


URL은 다음과 같은 형태로 이루어져 있다

스킴(프로토콜)    |    서버의 주소           |    리소스
   http://           |  www.ituntitle.com     |   login.html

이런식으로 http의 상호작용 단위(트랜잭션)은 클라이언트의 http 요청 명령과 서버의 http 응답 결과로 이루어져있다

클라이언트와 서버 사이에 존재하는 중개사를 프락시라고 한다
주로 보안을 위해서 사용되며 필터작업 등을 한다

자주 요청되는 리소스들을 캐시라는 곳에 저장해서 빠르게 꺼낼 수 있도록 한다
